# Repository Guidelines

## プロジェクト構成とモジュール配置

このリポジトリは、記事ネタ収集とMarkdown下書き作成を行うローカルWebアプリです。

- `server.js`: Node.js標準機能だけで動くHTTPサーバー、API、Markdown保存処理。
- `public/`: ブラウザ画面のHTML、CSS、JavaScript。
- `articles/`: 生成・保存されたMarkdown記事。必要なものだけ管理対象にしてください。
- `assets/`: 今後追加するメディア、テンプレート、サンプル入力。
- `docs/`: コントリビューター向け、またはユーザー向けの文書。
- `tests/`: 今後追加する自動テスト。

生成物、キャッシュ、エディタ固有のファイルは、意図して管理する場合を除きコミットしないでください。

## ビルド・テスト・開発コマンド

現時点では外部依存関係はありません。Node.js 24系で動作確認しています。

- `node server.js`: ローカルWebアプリを起動する。
- `npm run dev`: `node server.js` と同じ起動コマンド。
- `npm start`: 本番相当の起動コマンド。
- `node --check server.js`: サーバーコードの構文チェック。
- `node --check public/app.js`: フロントエンドコードの構文チェック。

PowerShellの実行ポリシーにより `npm` が使えない環境では、`node server.js` を使ってください。

## コーディングスタイルと命名規則

導入した言語やフレームワークの慣習に従ってください。現在はプレーンなJavaScript、HTML、CSSで構成しています。

ファイル名とディレクトリ名は説明的にしてください。Webアセットやスクリプトには、`content-parser.ts` や `editor-toolbar.css` のような小文字の kebab-case を推奨します。コードシンボルは言語標準に合わせ、JavaScript変数は `camelCase`、Reactコンポーネントを導入する場合は `PascalCase` を使ってください。

## テスト方針

テストフレームワークはまだ設定されていません。新機能を追加する際はテストも追加し、`collects-news-ideas` や `saves-markdown-draft` のように振る舞いが分かる名前にしてください。

フレームワークを選定したら、次を記載してください。

- 全テストの実行方法。
- 単一テストファイルの実行方法。
- 重要モジュールに求めるカバレッジ。

中核的な振る舞いの確認を、手動検証だけに頼らないでください。

## コミットとプルリクエストの方針

このディレクトリは現在Gitリポジトリとして初期化されていないため、既存のコミット履歴から規約は確認できません。プロジェクト固有の規約が決まるまでは、`Add article draft workflow` や `Fix news idea collection` のように、簡潔な命令形の件名を使ってください。

プルリクエストには、概要、変更理由、テスト結果、UI変更がある場合はスクリーンショットや録画を含めてください。関連Issueがあればリンクし、残作業や制限事項も明記してください。

## エージェント向け指示

編集前に現在のファイルを確認してください。初期開発では構成が短期間で変わる可能性があります。変更範囲は絞り、無関係な整形差分は避けてください。ビルドコマンド、テストツール、リポジトリ構成を導入したら、このガイドも更新してください。
